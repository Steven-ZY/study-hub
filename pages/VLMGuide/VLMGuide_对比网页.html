推理时引导 VLA 方法对比 · VLMGuide 方案规划

聚焦"不重训策略、在推理时引导冻结 VLA 解决 OOD"这条研究线 —— 它们的数据流、引导机制，以及我们的差异化定位 · 2026-06
一句话主线：当部署的 VLA（如 π0.5）遇到训练分布外的新物体时会失败。重训成本高，于是一批工作转向推理时引导——不改策略参数，在动作生成过程注入引导信号纠偏。本页对比它们「怎么生成引导信号」「怎么注入策略」，并指出它们的共同盲区与我们的切入点。

二、它们各自怎么做引导？（数据流 + 机制）

DISCO 2024

VLM keyframe + 约束 inpainting
语言+多视图 → VLM 出 3D keyframe(粗目标) → 反向去噪每步: min 距keyframe s.t. 似然≤γ → 投影到"贴近目标且在分布内"
引导=投影：把动作投影到"既贴近 keyframe、又留在训练分布高似然区"的集合。keyframe 不准时靠固定阈值 γ 被动拉回。

SuSIE 2023

图像编辑生成 subgoal + 条件执行
当前图+语言 → InstructPix2Pix 编辑出"未来 subgoal 图" → goal-conditioned policy π(a | 当前, subgoal) 直接够到 → 每 k 步刷新 subgoal
引导=条件执行：生成目标图后训一个专门的低层策略去够到它。生成即信，不检验可达性（作者自承是 future work）。

GHIL-Glue 2024

SuSIE + 过滤坏 subgoal
生成 K 个候选 subgoal → 分类器 f(s,g,l) 打分 (ResNet+语言, 判"是否导向进展") → argmax 选 top-1, 其余丢弃 → 喂 goal-conditioned policy
引导=过滤+执行：用任务进展分类器挑最好的候选。但是二元丢弃、看不出"可达性"、底层仍是条件执行（明确放弃了梯度引导路线）。

DynaGuide 2025

动态模型梯度引导（本方案骨架）
人工给定 g+/g- 图像 → 动态模型 f(o,a)→未来 latent → 算到 g+/g- 的距离 → ∇_a 求梯度, 注入 DDIM 去噪 → "推"动作走向 g+ 避开 g-
引导=梯度：独立动态模型预测动作后果，对目标距离求梯度引导冻结策略。目标人工给定、开环、假设引导永远可信。

VLS 2026

VLM 写 reward 代码 + 几何势场
OOD 输入 → SAM+DINOv2+深度 → 3D keypoint → VLM 写可微 PyTorch reward R_s → ∇R_s 注入去噪 + FK 重采样 + RBF → 闭环 Schmitt 切 stage
引导=梯度：VLM 直接生成 reward 函数（几何势场），无独立动态模型。无负目标、无可信度收手，自适应 λ 基于相对 reward。

OmniGuide 2026

3D 能量场统一多源引导
VLM点/3D模型/人姿 → 统一为 3D 空间 吸引子+排斥子 → 能量 L(X), ∇_X L → 经可微运动学 backprop 到动作 → 注入 flow matching(π0.5/GR00T)
引导=梯度：把一切引导"几何化"为 3D 吸引/排斥能量场。语义也先 collapse 成一个 3D 点。恒定强度、开环、无可信度建模。

共同点（赛道已占满的部分）：都冻结策略、都在去噪/采样过程注入引导、都用 VLM/外部模型产目标、都做正负两极。共同盲区：全都假设引导信号可信，没有检验"目标是否可达/是否幻觉"。

三、我们的方案：VLMGuide

核心主张：现有方法都默认引导目标可信，但真实 OOD 下 VLM 会幻觉、生成目标会不可达，盲目引导反而有害。我们把"目标可达性/可信度检验"作为一等公民——用视觉动力学判断目标到底够不够得到，不可达就自适应收手或闭环重生成。
这正是 SuSIE、GHIL-Glue 等代表作公开承认、但都未做的方向。

冻结 π0.5 flow matching
观测 I_t + 任务 l → VLM 生成图像编辑指令（+失败诊断） → 图像编辑模型 出 goal 图 I_goal（RGB 给人/VLM 看；编码到 DINOv2 latent 求梯度） → 视觉动力学 f(φ(I_t), a) → 预测未来 latent → 可达性 c = 动作空间里能否够到 φ(I_goal)? ← 核心 可达 → α(c) 正常引导 不可达/幻觉 → 减弱引导 / 收手 / 让 VLM 重新编辑 → classifier guidance 注入 π0.5 采样 → 执行 → 闭环诊断

循序渐进的分阶段路线（不一上来做全套）

S0 复现梯度引导基线（给定目标）
S1 接图像编辑生成目标
S2 ★ 加可达性检验（关键验证点）
S3 不确定性自适应引导强度
S4 VLM 闭环诊断+重编辑+负目标
S5 松灵双臂真机·contact-rich

每阶段独立验证、出一个结果再进下一步。S0→S1→S2 是地基，S2（可达性检验）是 go/no-go 卡点——做不出正向结果就止损，避免盲目投入。

四、与已有工作的区别（四条护城河）

① 可达性 vs 任务进展分类
GHIL-Glue 用分类器判"是否导向进展"，看不出物理可达性；我们用视觉动力学判"动作究竟够不够得到"。且这是 GHIL-Glue 自己点名的 future work。
② 梯度引导冻结 VLA vs 条件执行
SuSIE/GHIL-Glue 训专门低层策略去够目标；我们直接梯度引导冻结的 π0.5，享受大模型能力，不训低层。
③ 连续自适应 vs 二元过滤
它们对坏目标"丢弃/不丢弃"二选一；我们按可信度连续调节引导强度、不确定时收手。
④ 闭环重编辑 vs 生成即信
它们生成目标后直接用；我们让 VLM 根据失败诊断重新生成目标，形成闭环纠错。

关键差异化实验

构造"语义对但动力学不可达"的场景 —— VLM/图像编辑生成的目标语义正确，但当前状态下物理够不到：
SuSIE（直接执行） → 失败 GHIL-Glue（任务进展分类）→ 误判为好目标, 仍失败 VLMGuide（可达性检验） → 识别不可达 → 收手/重编辑 → 成功
一个实验直接证明"可达性引导 > 任务进展分类过滤"，是区别于同类工作的最强证据。
